Logic Gates

Austin Fay January 14, 2018

Computers are just a bunch of cleverly arranged transistors. But how do transistors actually perform computational operations? The answer is through boolean logic. Boolean logic is simply the idea that we can make logical conclusions from variables with only two states. Those states could be represented as false/true, 0/1, yin/yang, 0 volts/5 volts, you name it. Each logic gate has a specific output for a set of given inputs, which you will see later.

Great, so there are some logic gates that we can supposedly use to build a computer. But how do we make these out of electrical components? Well, keep scrolling and you might find out.

Buffers

00
11

Buffers are the simplest type of logic gate. They only have one input and one output. You can see a table to the right of all of the possible inputs and outputs of a buffer. We call a table like this a truth table. When we come up with more complicated circuits, truth tables help us identify what outputs we want from a given set of inputs.


To the right you can see the Logisim circuit that makes up a buffer. In the center we have our transistor. Connected to the base of our transistor is our input. Our source of power is connected to the collector of the transistor, and the output is connected to the emitter. Our input will dictate whether or not current flows from the collector to the emitter. If our input is 0, there is no current flowing from the collector to the emitter, so the voltage potential is set to low with the pull-down resistor connected to the output. This makes our output 0. If our input is 1, current flows from power to our output. Since only a small amount of current can flow through our pull-down resistor compared to our transistor, the voltage potential is high meaning we have a 1 at our output. Once we build our circuit and we know that it will work consistently how we want to, we can represent it as a triangle (shown to the left of this section's header), and move up a layer of abstraction.

Okay, so we've built a buffer before. But how do we do real logic? Well, to the right you can see the truth table for an AND gate. As the name implies, we only want an output of 1 if Input A and Input B are 1. In any other case, our output should be 0.


To the right is the Logisim circuit for an AND gate. As you can see, the circuit looks a lot like two buffers in series. You would be correct in making that observation. If both of the inputs are zero, the output is zero by default as set by the pull-down resistor. If we set the first input as 1, current can flow through the first transistor but is stopped by the second transistor. If the second input is 1 but not the first, current doesn't flow through either of the transistors. It's not until we set both of the inputs to 1 when the current can flow through both of the transistors and reach the output.

AND gates are great and all, but what if we want to be less picky? Instead of requiring both inputs to be 1, what if we only want to require one input to be 1? Well in that instance, you might want to build an OR gate. The output of an OR gate will be 1 if either Input A or Input B is 1 as shown in the truth table to the right.


Unlike our AND circuit, which was constructed from putting two transistors in series, OR gates are constructed from putting two transistors in parallel. By putting the transistors in parallel, the current can bypass the other transistor and reach the output if at least one of the inputs is set to 1, meaning the output can be 1 if either of the inputs are 1.. If both inputs are set to 0, no current flows to the output, making the output 0. If both inputs are 1, current flows to the output through both transistors, making the output 1.
